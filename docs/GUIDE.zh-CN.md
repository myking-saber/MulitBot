# MulitBot 使用指南

## MulitBot 是什么？

MulitBot 是基于 [Claude Code](https://claude.ai/claude-code) 的多智能体协作框架。你用手机发需求，AI 老板自动组建专家团队（产品经理、架构师、设计师、测试等），他们讨论、规划、开发，你在手机上实时监控进展。

**简单来说**：你是客户，给老板发消息，老板招人干活，你看结果。

## 为什么需要 MulitBot？

| 痛点 | MulitBot 方案 |
|------|---------------|
| 一个 AI 做不好所有事 | 多个专业角色各司其职，有独立性格和专业能力 |
| AI 不讨论就直接写代码 | 强制讨论流程：需求 → 设计 → 技术方案 → 决策 → 编码 |
| 看不到项目进展 | 手机实时查看团队讨论，频道隔离，通知推送 |
| 长对话丢失上下文 | 三层记忆：个体记忆 + 团队知识 + 跨项目经验 |
| 闲置资源浪费 | 按需启动：初始只有老板，团队成员用到时才拉起 |

## 核心功能

### 客户端（你）
- **手机操作**：用手机浏览器发需求、查看讨论
- **实时通知**：@你 时声音、振动、浏览器推送
- **项目隔离**：每个项目独立频道（#proj-xxx）
- **一键中断**：随时停止正在进行的项目
- **导出纪要**：下载完整对话记录为 Markdown

### AI 团队
- **强制讨论**：不能跳过 → 需求 → 设计 → 技术 → 决策 → 编码
- **角色复用**：跨项目积累经验，越用越强
- **按需启动**：只在需要时启动 Claude 实例
- **崩溃恢复**：Monitor 自动检测并重启崩溃的 Bot
- **防循环**：两人对话不超过 3 轮，超了 Lead 介入

### 开发者
- **自托管**：所有数据在你的机器上，不依赖第三方服务
- **结构化日志**：Hub、Monitor、MCP 插件、系统事件全部 JSON 格式
- **多语言**：脚本、Web UI、AI 指令支持中英日三语
- **MIT 开源**

## 快速开始

### 环境要求

| 工具 | 版本 | 安装 |
|------|------|------|
| Bun | 1.0+ | [bun.sh](https://bun.sh) |
| Claude Code | 2.0+ | [claude.ai/claude-code](https://claude.ai/claude-code) |
| tmux | 3.0+ | `apt install tmux` |
| jq | 1.6+ | `apt install jq` |

需要 Claude 订阅（Claude Max 或 API Key）。

### 安装

```bash
git clone https://github.com/anthropics/MulitBot.git
cd MulitBot
cd plugins/multibot-hub && bun install && cd ../..
```

### 配置语言（可选）

编辑 `team.json`：
```json
{
  "lang": "zh-CN"
}
```

留空 `""` 则自动检测系统语言。

### 启动

```bash
bash scripts/start.sh
```

只启动 3 个进程：
- **Hub** — 消息服务器
- **Boss** — AI 老板
- **Monitor** — 自动唤醒 + 崩溃恢复

手机打开 `http://<你的IP>:7800`。

### 第一个项目

1. 在手机输入：`@boss 帮我做一个贪吃蛇游戏`
2. 看 Boss 自动组建团队（Lead + 产品 + 架构 + 美术）
3. 切换到 #proj-snake 频道看团队讨论
4. 收到完成通知
5. 代码在 `projects/snake/code/`

### 停止

```bash
bash scripts/stop.sh
```

## 远程访问

### 同一 WiFi
直接打开 `http://<局域网IP>:7800`

### 带密钥
```bash
HUB_SECRET=你的密钥 bash scripts/start.sh
# 手机: http://<局域网IP>:7800?secret=你的密钥
```

### 公网（cloudflared 隧道）
```bash
HUB_SECRET=你的密钥 HUB_TUNNEL=cloudflared bash scripts/start.sh
```

## 记忆系统

```
项目 1                      项目 2
┌────────────┐              ┌────────────┐
│ architect   │──────────────│ architect   │  ← 同一角色复用
│ 记忆: v1    │   经验文件   │ 记忆: v2    │
│             │──────────>   │ + v1 经验   │  ← 经验自动带入
└────────────┘              └────────────┘

workspace/team-knowledge/    workspace/team-knowledge/
（项目内共享）                （各项目独立）
```

- **个体记忆**：Claude Code 自动保存在 bot 目录
- **团队知识**：`workspace/team-knowledge/` 共享文件
- **跨项目经验**：`roles/<id>.experience.md` — 上下文压缩前保存，复用时自动载入

## 脚本说明

| 脚本 | 用途 |
|------|------|
| `start.sh` | 启动系统（Hub + Boss + Monitor） |
| `stop.sh` | 停止所有进程 |
| `add-member.sh` | 运行中添加团队成员 |
| `recruit.sh` | 生成/复用角色定义 |
| `list-roles.sh` | 列出可复用角色 |
| `monitor.sh` | 消息监控 + 自动唤醒 + 崩溃恢复 |
| `wake.sh` | 手动唤醒指定 Bot |
| `export-log.sh` | 导出频道对话为 Markdown |
| `debrief.sh` | 项目复盘 — 收集各角色经验总结 |
| `i18n.sh` | 脚本多语言库 |
